# Backend (FastAPI Microservice)

Привет! Эта папка `backend` работает абсолютно изолированно от папки `frontend`. 
Здесь живет сервер обработки данных (API), который принимает заявки от интерфейса сайта, сохраняет их в базу данных и отправляет уведомление в Telegram.

## 📌 Как это работает (Простым языком)
1. Клиент на сайте жмет "Отправить заявку". Фронтенд шлет данные POST запросом на адрес `http://127.0.0.1:8000/api/leads`.
2. В файле **`main.py`** запрос ловится и направляется в роутер **`routers/leads.py`**.
3. Файл **`schemas.py`** проверяет, все ли поля (Возраст, Телефон и т.д.) правильно заполнены. Если хакер или спам-бот шлет сломанные данные — этот файл мгновенно сбросит соединение.
4. Файл **`models.py`** — это чертеж таблицы в Базе Данных (сейчас это SQLite, в будущем будет PostgreSQL). 
5. Код из роутера берет данные, сохраняет их в базу (`database.py`) и просит файл **`services/telegram.py`** отправить сообщение вам в чат. Секретные ключи хранятся в файле `.env`.

## 📂 Структура файлов
```text
leads_service/
 ├── .env                <- ВАЖНО: Тут лежат ключи (Telegram Bot Token, Chat ID). Не пуш в Github!
 ├── main.py             <- Основной мотор сервиса. Подключение CORS, объединение всех роутов.
 ├── database.py         <- Настройки подключения к базе данных.
 ├── models.py           <- Объекты ORM (таблица базы данных). 
 ├── schemas.py          <- Валидатор данных (Pydantic схемы).
 ├── routers/
 │   └── leads.py        <- Обработчики запросов (эндпоинты), например `/api/leads`.
 └── services/
     └── telegram.py     <- Код отправки сообщений в Telegram (через httpx).
```

## 🛠 Как запустить локально (инструкция)

**1. Установка и зависимости**
Нужно создать виртуальное окружение, чтобы не засорять ваш компьютер глобальными пакетами:
```bash
cd backend/leads_service
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

**2. Запуск сервера**
Запускать стоит через `uvicorn`. Он автоматически будет делать рестарт при изменении файлов:
```bash
uvicorn main:app --reload
```
Теперь API запущено на `http://localhost:8000`. 
Вы можете открыть `http://localhost:8000/docs` в браузере, чтобы увидеть готовую и красивую панель (Swagger UI), через которую можно тестировать отправку заявок без входа на сайт!

## 🔄 Миграция на PostgreSQL
Когда вы захотите перейти на PostgreSQL, вам будет достаточно запустить Docker с PostgreSQL, а в файле `database.py` поменять SQLite урл `sqlite+aiosqlite:///./leads.db` на ваш новый урл `postgresql+asyncpg://user:pass@localhost/db`. Всё остальное `SQLAlchemy` сделает сама!
